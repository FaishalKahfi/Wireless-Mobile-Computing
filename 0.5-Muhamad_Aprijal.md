Manajemen Proses pada Mobile OS (Android)
(Wireless & Mobile Computing – Modul OS)
1. Konsep Dasar Mobile OS
Mobile OS (seperti Android) dirancang berbeda dengan desktop OS karena:
•	Resource terbatas (RAM, CPU, baterai) 
•	Selalu terhubung jaringan (Wi-Fi, seluler) 
•	Berbasis sensor (GPS, kamera, dll) 
•	Harus hemat daya (power-aware system) 
________________________________________
2. Kategori Proses (Process States di Android)
Android mengelompokkan aplikasi berdasarkan tingkat kepentingannya:
1. Foreground Process
•	Aplikasi yang sedang digunakan langsung oleh user 
•	Contoh: membuka WhatsApp, kamera aktif 
•	Prioritas: tertinggi (tidak boleh dimatikan) 
2. Background Process
•	Aplikasi tidak terlihat, tapi masih berjalan 
•	Contoh: Spotify jalan di background 
•	Digunakan untuk multitasking 
•	Bisa dihentikan jika memori rendah 
3. Cached Process
•	Aplikasi yang sudah tidak aktif, hanya disimpan di RAM 
•	Tujuan: mempercepat buka ulang 
•	Contoh: aplikasi yang baru saja ditutup 
•	Prioritas: paling rendah (pertama di-kill) 
👉 Ini yang ditulis di papan:
Foreground → Background → Cache (urutan prioritas)
________________________________________
3. Inter-Process Communication (IPC)
Karena tiap aplikasi Android berjalan terpisah (sandbox), maka komunikasi dilakukan lewat IPC.
Fungsi IPC:
•	Pertukaran data antar aplikasi/proses 
•	Contoh: 
o	Share foto dari Galeri ke WhatsApp 
o	Buka Maps dari aplikasi lain 
Teknologi IPC di Android:
•	Binder (utama di Android) 
•	Intent (untuk komunikasi antar aplikasi) 
________________________________________
4. Workload Manager (Manajemen Beban Kerja)
Sistem mengatur:
•	CPU usage 
•	RAM usage 
•	Scheduling proses 
Tujuannya:
•	Menjaga performa tetap stabil 
•	Menghindari lag 
•	Menyesuaikan dengan kondisi device 
Contoh di HP kamu (Vivo Y33s):
•	Saat main game → CPU diprioritaskan 
•	Saat banyak app → RAM dibagi otomatis 
________________________________________
5. Keamanan Data (Security & Sandbox)
Di papan ada tulisan “keamanan data” & “user-driven apps”
Konsep:
•	Setiap aplikasi berjalan di sandbox (isolasi) 
•	Tidak bisa akses data aplikasi lain tanpa izin 
Contoh:
•	WhatsApp tidak bisa baca data Instagram 
•	Aplikasi harus minta permission: 
o	Kamera 
o	Lokasi (GPS) 
o	Storage 
________________________________________
6. Interaksi dengan Hardware & Sensor
Di papan ada:
“Interaksi → GPS, Wi-Fi, dll”
Artinya:
•	Aplikasi tidak akses hardware langsung 
•	Harus lewat Application Framework 
Contoh:
•	Google Maps → akses GPS 
•	YouTube → akses internet (Wi-Fi/data) 
•	Kamera → akses sensor kamera 
________________________________________
7. Optimasi Sistem (Optimization)
Di papan ada:
“Optimasi → aktif / pasif apps”
a. Active Apps
•	Sedang digunakan 
•	Dapat resource penuh 
b. Passive Apps
•	Tidak aktif 
•	Dibatasi (CPU, network, dll) 
________________________________________
8. Manajemen Baterai (Battery Management)
Mobile OS sangat fokus ke efisiensi daya:
Teknik:
•	Membatasi background process 
•	Doze Mode (Android) 
•	App Standby 
Contoh di Vivo Y33s:
•	Aplikasi jarang dipakai akan “dibekukan” 
•	Notifikasi tetap jalan, tapi hemat baterai 
________________________________________
9. Kesimpulan
Mobile OS seperti Android dirancang untuk:
•	Mengatur proses berdasarkan prioritas (foreground, background, cache) 
•	Mengontrol komunikasi antar aplikasi (IPC) 
•	Menjaga keamanan dengan sandbox 
•	Mengoptimalkan performa & baterai 
•	Mengelola interaksi dengan hardware (GPS, Wi-Fi, sensor) 
Semua ini bertujuan agar:
👉 HP tetap cepat, aman, dan hemat daya meskipun banyak aplikasi berjalan
