1. Teknologi Natif (Native App)
Point (Gagasan Utama): Pengembangan Native adalah pilihan mutlak jika prioritas utama sistem adalah performa komputasi maksimal dan akses tak terbatas ke perangkat keras (hardware), meskipun membutuhkan investasi yang besar.
Reason (Alasan): Aplikasi dibangun secara spesifik menggunakan bahasa pemrograman dan tools resmi dari sistem operasi (misalnya Java/Kotlin untuk Android, Swift untuk iOS). Karena tidak ada "lapisan penerjemah" di tengahnya, aplikasi bisa berkomunikasi langsung dengan inti sistem operasi. Hal ini memberikan kecepatan tertinggi, namun menuntut sumber daya spesialis untuk setiap platform.
Example (Contoh/Bukti di Papan): Sesuai matriks, Native memiliki akses hardware secara penuh ("full NFC, Cam") dan performa di tingkat "Height" (Tinggi). Konsekuensinya, biaya (Biaya Tim) juga "Height" karena butuh pengembangan terpisah untuk "3x OS" (Android, iOS, dan Web). Distribusinya menggunakan jalur resmi seperti "PlayStore, App Store".
Point (Kesimpulan): Kesimpulannya, arsitektur Native adalah "Pilihan tepat" untuk proyek kompleks yang menuntut performa tinggi dan sensor intensif, seperti "Game AR/VR".
2. Teknologi Hibrida (Hybrid App)
Point (Gagasan Utama): Pendekatan Hybrid menawarkan jalan tengah (sweet spot) yang strategis antara efisiensi biaya pengembangan lintas-platform dan kemampuan memanfaatkan fitur dasar ponsel.
Reason (Alasan): Developer cukup menulis satu basis kode (biasanya menggunakan teknologi web seperti JavaScript) yang kemudian "dibungkus" agar bisa diinstal di berbagai OS. Untuk mengakses fitur perangkat keras, Hybrid menggunakan plugin sebagai jembatan. Ini sedikit memangkas performa dibandingkan Native, tetapi sangat menghemat waktu dan biaya tim.
Example (Contoh/Bukti di Papan): Di papan tercatat bahwa akses hardware mengandalkan "Plugin" (seperti Cordova atau React Native). Performanya berada di tingkat "M" (Medium), begitu pula biaya timnya yang "M" karena efisiensi basis kode "2x P+W" (Platform + Web). Distribusinya tetap bisa masuk ke store "Masing-masing".
Point (Kesimpulan): Oleh karena itu, Hybrid menjadi "Pilihan tepat" untuk membangun sistem internal bisnis perusahaan yang butuh diinstal di ponsel pegawai dengan fitur standar, seperti "App Web, ERP, POS" (Point of Sale).
3. PWA (Progressive Web App)
Point (Gagasan Utama): PWA adalah arsitektur yang paling lincah, efisien, dan cepat didistribusikan, dirancang khusus untuk menjangkau pengguna seluas mungkin tanpa hambatan instalasi aplikasi tradisional.
Reason (Alasan): PWA pada dasarnya adalah situs web tingkat lanjut yang memanfaatkan teknologi modern (seperti Service Workers) agar bisa berfungsi menyerupai aplikasi mobile (bisa di-install di homescreen dan bekerja saat offline). Karena sepenuhnya berjalan di atas standar web, PWA bebas dari birokrasi app store, namun akses perangkat kerasnya dibatasi oleh standar keamanan browser.
Example (Contoh/Bukti di Papan): Catatan menunjukkan PWA hanya mengandalkan "Web Standard API" untuk perangkat keras dan "Worker, JSON" untuk operasi dan pertukaran data. Keunggulan mutlaknya ada di biaya tim yang sangat "Low" karena hanya butuh satu tim "Web" konvensional. Distribusinya dilakukan secara instan langsung melalui URL/ "Web".
Point (Kesimpulan): Singkatnya, PWA adalah "Pilihan tepat" jika target utamanya adalah aksesibilitas universal di "All OS" (semua OS dan perangkat) dengan anggaran pengembangan yang minimalis, seperti sistem informasi akademik atau sistem manajemen lab berbasis web.
